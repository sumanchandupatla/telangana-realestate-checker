# Testing the Search Flow

## Architecture

The app serves both frontend (static Next.js export) and API from the same FastAPI server on port 8000:
- Frontend: static files in `backend/static/` served via catch-all route
- API: `/api/unified/search`, `/api/rera/search`, `/api/dharani/search`, `/api/bpass/search`, `/api/hmda/search`
- Frontend fetches use relative paths (e.g. `/api/unified/search`) — no CORS issues

## Starting the Backend

```bash
cd /home/ubuntu/repos/telangana-realestate-checker/backend
pip install uvicorn fastapi httpx beautifulsoup4 lxml
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

If frontend static files are missing from `backend/static/`, rebuild:
```bash
cd /home/ubuntu/repos/telangana-realestate-checker/frontend
npm install
npm run build
cp -r out/ ../backend/static/
```

## Exposing for Browser Testing

Use `deploy expose --port 8000` to get a public tunnel URL. The tunnel uses HTTP Basic Auth — the browser will show a popup for credentials.

## Key Assertions

1. **Search returns results**: After searching a valid RERA number, verify the result card shows project name, promoter, status badge (green = Registered), and source (TS-RERA)
2. **No connection errors**: Invalid queries should show "No results found for X" — NOT "Failed to connect to backend". The latter indicates a fetch/CORS issue.
3. **Console clean**: No errors containing "credentials", "CORS", "blocked", or "Mixed Content" in browser console
4. **Loading state**: During search, should show "Querying government portals..." with animated spinner

## Known Behaviors

- HMDA DPMS frequently times out — this produces a warning banner "Some portals returned warnings" with "HMDA: HMDA DPMS timed out." This is expected and not a bug.
- The unified search (`/api/unified/search`) queries all portals in parallel. Total response time is bounded by the slowest portal (usually HMDA at ~15-20s timeout).
- TS-RERA is the most reliable portal and returns results quickly (~2-5s).

## Common Issues

- If search shows "Failed to connect to backend", check that `API_BASE` in `frontend/src/lib/api.ts` is set to `""` (empty string for same-origin) and NOT an absolute URL with credentials.
- If you see CORS errors, ensure frontend is being served from the backend (same origin), not from a separate static host.
- The frontend `.env.local` should have `NEXT_PUBLIC_API_URL=` (empty) for same-origin setup.

## Devin Secrets Needed

No secrets required for testing — the app queries public government portals. The tunnel auth credentials are auto-generated by the deploy tool.
